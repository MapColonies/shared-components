# Public API — @map-colonies/react-components

## Single entry point

All public types and components are exported from the package root. Never import from `dist/` or any subpath.

```ts
// Correct
import { CesiumMap, IRasterLayer, IMapLegend, GeocoderOptions } from '@map-colonies/react-components';

// Wrong — do not use dist subpaths
import { IRasterLayer } from '@map-colonies/react-components/dist/cesium-map/layers-manager';
import { IMapLegend }   from '@map-colonies/react-components/dist/cesium-map/legend';
```

## Barrel structure

```
src/index.ts                          ← package entry point
└── src/components/index.ts           ← aggregates all component modules
    ├── cesium-map/index.ts           ← CesiumMap + all cesium types
    │   ├── map.tsx                   → CesiumMap, IBaseMaps, IBaseMap, useCesiumMap, …
    │   ├── layers-manager.ts         → IRasterLayer, ICesiumImageryLayer, …
    │   ├── proxied.types.ts          → CesiumXxx proxy classes
    │   ├── legend/index.tsx          → IMapLegend, LegendSidebar, LegendWidget
    │   ├── geocoder/index.ts         → GeocoderOptions, GeocoderPanel, GeocoderWidget
    │   ├── layers/, entities/, …
    │   └── tools/
    ├── ol-map/index.ts               ← OpenLayers map
    ├── date-picker/                  ← date/time pickers
    ├── file-picker/                  ← file picker
    └── … (other components)
```

## How to expose a new public type

1. Define (or import) the type in its source file, e.g. `src/components/cesium-map/my-feature.ts`.
2. Add an `export` for it in the nearest barrel — e.g. `src/components/cesium-map/index.ts`.
   ```ts
   export type { MyNewType } from './my-feature';
   // or
   export * from './my-feature';
   ```
3. The root barrel (`src/index.ts → src/components/index.ts`) will pick it up automatically.
4. Rebuild (`yarn build`) to regenerate `dist/index.d.ts`.

**Do not** create a new top-level barrel file. Everything flows through `src/components/index.ts`.

## CSS / asset imports

CSS-only imports (not types) may still reference `dist/` where no alternative exists:

```ts
import '@map-colonies/react-components/dist/date-range-picker-with-presets/styles';
```

This is acceptable until a proper CSS export strategy is in place.

## Declaration file

Types are declared in `dist/index.d.ts` (generated by `vite-plugin-dts` with `rollupTypes: true`).
The `package.json#types` field points there. No manual declaration changes are needed.
